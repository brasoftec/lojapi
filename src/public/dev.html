Como integrar seu ERP
1
Gere um Token ERP
Vá em Tokens ERP → clique em Gerar token → copie o token gerado. Ele só aparece uma vez.
X-API-Key: tk-ot...sua-chave...bra
2
Configure o .env do ERP
Vá em Variáveis .env → copie o bloco pronto e cole no .env do seu ERP.
LOJAPI_BASE_URL=https://api.ofertatop.com.br/api/v1
LOJAPI_API_KEY=tk-ot...bra
LOJAPI_STORE_ID=uuid-da-loja
LOJAPI_STORE_SLUG=slug-da-loja
3
Teste a conexão
Confirme que o token está funcionando consultando os dados da loja.
curl https://api.ofertatop.com.br/api/v1/loja \
  -H "X-API-Key: $LOJAPI_API_KEY"
4
Configure o Webhook
Vá em Webhook → informe a URL do endpoint no seu ERP → salve → clique em Testar. Seu ERP deve responder HTTP 200.
POST /webhook/configurar
{"webhookUrl": "https://seu-erp.com/api/webhook/lojapi"}
5
Sincronize o catálogo
Envie os produtos do ERP para a loja via POST /produtos. Use PATCH /produtos/:id/estoque para atualizar estoque.
POST /api/v1/produtos
{"name":"Produto","price":99.90,"sku":"SKU-001","stock":50}
6
Receba pedidos via Webhook
Quando um pedido for criado na loja, seu ERP recebe o evento order.created automaticamente com todos os dados.
{"event":"order.created","data":{"orderNumber":"#000001","total":199.90,"customer":{...},"items":[...]}}
7
Atualize status dos pedidos
Conforme o ERP processa, atualize o status. Fluxo: PENDING → CONFIRMED → PROCESSING → SHIPPED → DELIVERED
PATCH /api/v1/pedidos/:id/status
{"status":"SHIPPED"}

PATCH /api/v1/pedidos/:id/pagamento
{"paymentStatus":"PAID","paymentMethod":"pix"}
8
Múltiplas lojas
Cada loja tem seu próprio token. Para gerenciar várias lojas no mesmo ERP, use um token diferente por loja e identifique pelo LOJAPI_STORE_ID.
# Loja A
LOJA_A_API_KEY=tk-ot...bra
LOJA_A_STORE_ID=uuid-loja-a

# Loja B
LOJA_B_API_KEY=tk-ot...bra
LOJA_B_STORE_ID=uuid-loja-b
